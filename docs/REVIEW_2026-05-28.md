## 3. What to prioritise next

In rough order, biggest impact first.

1. 🟡→🟢 **Mobile layout pass** — touch every page on a 360×800 viewport, fix the timed-exam grid, dark-toggle position, and any sidebar overflow. Test with Playwright's `browser_resize`.
2. 🟡→🟢 **CI for tests + ruff** — 30-line GitHub Actions workflow that runs `pytest` + `ruff check` + `ruff format --check` on push. Cheapest single thing that lifts code-quality colour from yellow to solid green over time.
3. 🟡→🟢 **Spaced repetition for missed/weak** — schema add of `next_review_at`, `ease_factor`, `interval_days` to `question_stats`, SM-2 update on every `record_answer`. Surfaces "X questions due today" on Home. This is the single biggest UX uplift for cert-prep.
4. 🟡→🟢 **Study plan + daily goal** — picker on Account: "target exam date + target score". Home shows "Day 14/30, 47 questions today to stay on track". No new tech needed, just arithmetic on `question_stats`.
5. 🟡→🟢 **First-pass UX polish** — `st.balloons()` on first pass per cert, `st.dialog` confirmation on Quit, swap email→`profiles.username` in the welcome line, make recent-sessions rows clickable to a session-detail page.
6. 🟡→🟢 **Onboarding tour** — 4-step intro for first-time signed-in users (pick cert → first practice → review missed → check stats). One-time, dismissable, stored in `profiles.preferences`.
7. 🟡→🟢 **Per-question notes** — `bookmarks.note` column already exists in DB; just needs a textarea in the bookmark UI.

Items 1–4 above are the highest-leverage; 5–7 are quick wins to bank in the same sprint.
